# Implementaciones de referencia

Las implementaciones contenidas en esta carpeta constituyen la especificación funcional del proyecto.

Todos los algoritmos deberán ser instrumentados para producir una `ExecutionTrace`, pero su comportamiento no podrá modificarse.

Está permitido:

- agregar eventos;
- agregar identificadores de estado;
- registrar métricas.

No está permitido:

- alterar el orden de evaluación;
- optimizar la lógica;
- modificar la estrategia algorítmica;
- cambiar el recorrido de la DP;
- sustituir una implementación por otra equivalente.

Las visualizaciones, métricas y comparaciones dependen de que estas implementaciones permanezcan invariantes.
